# Acceptance Criteria Template

## Purpose
Acceptance criteria define the conditions under which a user story or requirement is considered complete and acceptable to stakeholders.

---

## Acceptance Criteria Format
Acceptance criteria should be:
- Clear
- Testable
- Unambiguous
- Aligned to business outcomes

---

## Example (Given / When / Then)

**User Story:** View customer interaction history

- Given the user is logged into the system  
- When the user opens a customer profile  
- Then the system displays all past interactions in chronological order  

- Given customer data is unavailable  
- When the user accesses the profile  
- Then the system displays a clear error message

---

## Non-Functional Acceptance Criteria
- Page loads within 3 seconds
- Data displayed is no more than 5 minutes out of sync
- Access is restricted based on user role

---

## Validation
Acceptance criteria are reviewed and approved by:
- Business stakeholders
- Product Owner / Delivery Lead
- QA / Testing team
