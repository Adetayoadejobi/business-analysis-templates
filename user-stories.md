# User Stories Template

## Purpose
This template is used to capture user needs in a clear, concise, and testable format that supports Agile and hybrid delivery environments.

---

## User Story Format
**As a** [type of user]  
**I want** [some goal or capability]  
**So that** [business value or outcome]

---

## Example User Story
As a Customer Service Advisor,  
I want to view customer interaction history in a single dashboard,  
So that I can respond to customer enquiries more efficiently and accurately.

---

## Additional Details
- Priority: High / Medium / Low
- Status: Draft / Approved / In Progress / Complete
- Dependencies: CRM integration, data availability
- Assumptions: User has appropriate system access

---

## Linked Acceptance Criteria
Refer to the acceptance criteria defined in `acceptance-criteria.md` to ensure the story is testable and aligned with business expectations.
